## AutoDL 启动步骤

### Step 1: 上传文件
```bash
# 在 AutoDL 终端执行
mkdir -p /root/autodl-tmp/DeepCiteFact
cd /root/autodl-tmp/DeepCiteFact

# 上传后解压
tar -xzf DeepCiteFact.tar.gz
```

### Step 2: 下载模型（如果用 API，跳过这步）

**如果用 API 替代 Verifier：**
```bash
# 只需要下载 Actor 模型（Qwen3-8B）
mkdir -p /root/autodl-tmp/models
cd /root/autodl-tmp/models

# 方法 1: HuggingFace（如果能连）
git lfs install
git clone https://huggingface.co/Qwen/Qwen3-8B

# 方法 2: ModelScope（国内更快）
pip install modelscope
modelscope download Qwen/Qwen3-8B --local_dir ./Qwen3-8B
```

**如果本地部署 7B Verifier（不用 API）：**
```bash
# 额外下载 Qwen2.5-7B-Instruct
git clone https://huggingface.co/Qwen/Qwen2.5-7B-Instruct
# 或
modelscope download Qwen/Qwen2.5-7B-Instruct --local_dir ./Qwen2.5-7B-Instruct
```

### Step 3: 安装依赖
```bash
cd /root/autodl-tmp/DeepCiteFact/verl
pip install -e .

# 安装 vllm（先装这个）
pip install vllm==0.5.4

# 安装 sglang
pip install "sglang[all]==0.4.9.post6"

# 如果用 API，安装 requests

```

### Step 4: 启动 Verifier 服务（仅本地部署需要，API 方案跳过）

**终端 1（CUDA_VISIBLE_DEVICES=0,1）：**

```bash
--model /root/autodl-tmp/models/Qwen2.5-7B-Instruct \
--host 0.0.0.0 \
--port 8000 \
--tensor-parallel-size 2 \
--trust-remote-code \
--gpu-memory-utilization 0.9 \
--max-num-seqs 256
```

**终端 2（CUDA_VISIBLE_DEVICES=2,3）：**
```bash

--model /root/autodl-tmp/models/Qwen2.5-7B-Instruct \
```

## 成本估算

| 配置 | Verifier 方案 | 卡数 | 单价/小时 | 预计时长 | 总成本 | 适用场景 |
|------|--------------|------|----------|---------|--------|----------|
| **API 方案（快速验证）** | SiliconFlow API | 4 | ¥3.5×4=¥14 | 2.5h | **¥35** (含 API 费) | 调试/原型验证 |
| **本地 7B（推荐）** | Qwen2.5-7B-Instruct | 6 | ¥3.5×6=¥21 | 1.5h | **¥32** | 完整训练 |
| 本地 32B | Qwen2.5-32B-Instruct | 8 | ¥12×8=¥96 | 1h | **¥96** | 追求最佳效果 |

**API 成本估算**：5000 条 × 8 samples × 2 turns × 200 tokens × ¥0.01/1k ≈ ¥16

⚠️ **注意**：API 方案训练时间更长（网络延迟），且可能遇到 rate limit。建议先用 API 跑 10 steps 验证流程，再切换到本地部署完成训练。

**建议：先租 6×4090 跑通流程（¥32），验证后再考虑上 A100**

## 常见问题

### Q1: 没有 SFT checkpoint 能直接跑吗？
**可以！** 直接用基座 Qwen3-8B 开始 RL 训练（只是收敛会慢一点）

### Q2: 训练到一半断了怎么办？
设置 `trainer.resume_mode="auto"`，会自动从最近 checkpoint 恢复

### Q3: 显存爆了怎么办？
降低 batch_size 或 max_length：
```yaml
data:
  train_batch_size: 32  # 从 64 降到 32
  max_response_length: 4096  # 从 8192 降到 4096
```

### Q4: Reward 一直是 0 怎么办？
检查 DDGS 搜索是否正常（Step 5），以及 Verifier 服务是否启动（curl http://127.0.0.1:8000/v1/models）
